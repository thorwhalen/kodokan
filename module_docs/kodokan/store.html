kodokan.store

dol-backed stores for the pipeline’s artifacts (the analysis SSOT).

Two dict-like stores keyed by video_id:

  • pose_store() — video_id -> PoseSequence, each persisted as a tidy/long Parquet file (columns fidx, frame, t_sec, person, keypoint, x, y, conf) with sequence metadata (fps, dims, backend, source_url, frame indices) carried in the Parquet schema metadata. Tidy/long Parquet is the analysis SSOT — columnar, compressed, and pandas/polars/DuckDB-native (research §6).

  • segments_store() — video_id -> list[demo dicts] as JSON.

load_all_tidy() concatenates every clip’s table (adding a video_id column) into one DataFrame for cross-clip analysis.
